## Part B: Instance-Scoped Cilium Policy Generation (implement in PlanEDC)

### Context

CCNP contracts only enforce class membership, not instance binding. 143 app-to-app service edges are authorized by Istio AuthorizationPolicies at L7 but have no Cilium L3/L4 equivalent. Any postgres-capable pod can reach every postgres server. This is not least privilege.

**Goal:** Generate instance-scoped Cilium policy from normalized Istio service edges, prove non-widening parity, then retire the broad class contracts.

### Generator Architecture

#### Input: Git-rendered Istio manifests (NOT live cluster)

Primary source: `kubectl kustomize` on the repo's Istio policy overlays. Reproducible, CI-safe, no cluster drift contamination.

Live cluster is used ONLY for:
- Selector cardinality validation (does selector match expected pods?)
- SA → pod label audit
- Post-deployment Hubble verification

#### Intermediate model: atomic edges

Do NOT translate 1 Istio policy → 1 Cilium policy directly. Normalize first.

Each Istio policy decomposes into atomic edges:
```yaml
- dest:
    namespace: hookshot
    selector: {app.kubernetes.io/name: tactical-backend}
  source:
    type: exact-sa
    namespace: hookshot
    serviceAccount: tactical-nginx
  ports:
    - port: "8080"
      protocol: TCP
  provenance: allow-tactical-nginx-to-tactical-backend
  translation: exact-sa
```

Then group only when grouping is provably lossless (same destination selector + same ports → merge source SAs into one fromEndpoints list).

#### Formalized intermediate representation: edgegraph/v1

Pipeline: `renderIstio() → normalizeEdges() → buildEdgeGraph() → validateSelectors() → auditSAs() → emitCNPs() → generateDiffs()`

This makes the generator a policy graph compiler — the EdgeGraph can later drive egress policies, firewall rules, or visualization.

#### Fail-closed on unsupported semantics

Generator MUST detect and REFUSE to translate:
- `when:` conditions (HTTP methods/paths/hosts/JWT)
- `to.operation.hosts`, `to.operation.methods`, `to.operation.paths`
- `requestPrincipals` / JWT conditions
- IP block conditions
- Negative matches
- Wildcard principals
- Rules with no ports (Istio intent broader than L4)
- Selectors relying on labels not present in Cilium endpoints

For these: emit nothing, write to gap report, exit nonzero in `--check` unless allowlisted.

#### Selector cardinality + drift validation

For every destination selector, resolve against live cluster:
- 0 matches → flag as error
- \>expected matches → flag as ambiguity warning
- Multiple distinct workloads sharing one selector → flag as multi-workload collision
- Record `selector_resolution.json` mapping selector → matched pods + actual stable labels (detects label drift)
- Allowlist headless service patterns: `cnpg.io/cluster`, `redis_setup_type`, galera clusters

#### Three-graph anti-widening proof

Three normalized graphs:
1. **Istio intent graph** — from Git-rendered Istio AuthorizationPolicies
2. **Generated Cilium graph** — from generator output
3. **Actual cluster enforcement graph** — from live `kubectl get cnp -A` + `kubectl get ccnp -A`

Normalize to canonical edge sets (srcNS, srcSA, destNS, destSelectorHash, port, protocol). Compare sets, NOT YAML.

Two diffs:
- **Istio → Generated** (translation correctness)
- **Generated → Actual** (deployment correctness — catches stale broad rules)

#### SA → pod identity audit

- SA with 0 pods → flag as phantom identity
- SA exists but not referenced by any pods → flag as dead policy
- Record `sa_identity_inventory.json` mapping SA → pod list

### Generator Outputs

**Per namespace — ONE FILE PER DESTINATION WORKLOAD** (not per edge):
- `cnp-instance-<destination-workload>.yaml`
- Deterministic rule ordering via stable canonical key: `namespace|sorted(selector)|port|srcNS|srcSA`

**Cluster-wide:**
- `instance-policy-inventory.json`
- `instance-policy-gaps.yaml` (structured: namespace, policy, reason)
- `sa_identity_inventory.json`
- `selector_resolution.json`
- `identity_usage.json` (before/after identity count)

**Provenance header:**
```yaml
# GENERATED FILE — DO NOT EDIT
# Source of truth: planedc k8s cilium generate-instance-policy
# Derived from Istio AuthorizationPolicy: hookshot/allow-tactical-nginx-to-tactical-backend
# Translation mode: exact-sa
# Non-widening status: verified
```

Namespace-only translations explicitly tagged as `translation: namespace-set` (broader than SA-exact).

### Identity Translation

| Istio | Cilium |
|-------|--------|
| `spec.selector.matchLabels` | `endpointSelector.matchLabels` |
| `source.principals: cluster.local/ns/<ns>/sa/<sa>` | `fromEndpoints: [{matchLabels: {k8s:io.cilium.k8s.policy.serviceaccount: <sa>}}]` |
| Cross-ns SA principal | Add `k8s:io.kubernetes.pod.namespace: <ns>` to fromEndpoints |
| `source.namespaces: [ns1, ns2]` | `fromEndpoints: [{matchExpressions: [{key: k8s:io.kubernetes.pod.namespace, operator: In, values: [ns1, ns2]}]}]` — tagged broader-than-SA |

### Modes

- `--generate` — write CNP files from rendered Istio overlay
- `--check` — diff against committed files, nonzero on drift or untranslatable gaps
- `--audit-cluster` — validate selectors against live cluster, report cardinality
- `--namespace <ns>` — scope to one namespace
- `--explain <policy-name>` — show translation trace for one policy
- `--visualize` — emit dot graph for visual debugging

### What the generator skips (owned by CCNPs)

default-deny, deny-default-sa, allow-dns, allow-gateway-ingress-*, allow-gatus-healthcheck, allow-cnpg-operator, allow-mariadb-operator, allow-redis-operator*, allow-prometheus-scrape, *-loadbalancer, allow-webhook-*, allow-intra-breakglass

If any skipped CCNP is broader than its corresponding Istio policy, log as intentional divergence in the gap report.
